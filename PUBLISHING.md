# Publishing

This package is intended to be published from GitHub Actions so that npm provenance is attached to every release.

## One-time setup

1. Create the npm package under the correct npm owner.
2. Configure a Trusted Publisher for this repository and workflow on npm:
   - Package: `@deepidealab/n8n-nodes-tracira`
   - Repository: `deepidealab/n8n-nodes-tracira`
   - Workflow file: `.github/workflows/publish.yml`
3. Confirm the repository is public.
4. Confirm the npm package metadata matches this repository and organization.

## Release flow

1. Update `package.json` version.
2. Update `CHANGELOG.md`.
3. Push the version change to `main`.
4. Create a GitHub release.
5. Let `.github/workflows/publish.yml` publish the package to npm with provenance.
6. The workflow waits for npm registry propagation and then runs `@n8n/scan-community-package` against the published package.

## Notes

- The publish workflow uses `id-token: write` for npm provenance and trusted publishing.
- The package must remain public for provenance to be generated by npm.
- Do not publish this package manually from a local machine if you want to stay aligned with n8n verification guidance.
- The n8n package scanner checks the published npm package, so an end-to-end scan can only run once the package exists on npm. Its ESLint pass, however, runs locally *before* publishing: `npm run scan:source` (scripts/scan-source.mjs) reproduces it against the working tree, and it is enforced in CI on every push and inside `npm run verify`, which gates the publish workflow. A violation fails the release before the version is burned on npm.
- `npm run lint` does not match the scanner. The scanner builds its own flat config and runs ESLint with `allowInlineConfig: false`, so **no `eslint-disable` comment has any effect** - not line-level, not block-level. A rule the scanner enforces has to be satisfied in the code itself. (This is what burned the 0.9.2 review: `npm run lint` was green while the scanner reported 9 errors, and the reviewer's own suggestion of a block-level disable comment also failed under `allowInlineConfig: false`.)
- The scanner is pinned as a devDependency. If a review ever flags something `scan:source` did not catch, update `@n8n/scan-community-package` first - the rules live in that package and n8n updates them over time.
- If migrating from the legacy unscoped package, deprecate `n8n-nodes-tracira` and point users to `@deepidealab/n8n-nodes-tracira`.
