# Publishing

This package is intended to be published from GitHub Actions so that npm provenance is attached to every release.

## One-time setup

1. Create the npm package under the correct npm owner.
2. Configure a Trusted Publisher for this repository and workflow on npm:
   - Repository: `deepidealab/n8n-nodes-tracira`
   - Workflow file: `.github/workflows/publish.yml`
3. Confirm the repository is public.
4. Confirm the npm package metadata matches this repository and organization.

## Release flow

1. Update `package.json` version.
2. Update `CHANGELOG.md`.
3. Push the version change to `main`.
4. Create a GitHub release.
5. Let `.github/workflows/publish.yml` publish the package to npm with provenance.
6. The workflow waits for npm registry propagation and then runs `@n8n/scan-community-package` against the published package.

## Notes

- The publish workflow uses `id-token: write` for npm provenance and trusted publishing.
- The package must remain public for provenance to be generated by npm.
- Do not publish this package manually from a local machine if you want to stay aligned with n8n verification guidance.
- The n8n package scanner checks the published npm package, so it can only pass after the package exists on npm.
